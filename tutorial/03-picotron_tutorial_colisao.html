Tutorial de colisão no Picotron — círculo, retângulo, mapa e técnicas avançadas

Picotron Colisão detecção e resposta
1

Conceitos fundamentais

Colisão em jogos 2D tem duas etapas distintas: detecção (os objetos se tocam?) e resposta (o que acontece quando tocam?). A técnica certa depende do formato dos objetos.

Os três formatos mais usados no Picotron:

círculoProjéteis, personagens, inimigos redondos. Colisão via distância — simples e barato.
AABBAxis-Aligned Bounding Box — retângulo sem rotação. O mais comum para personagens e itens.
tile sólidoCélulas do mapa marcadas com flag 0. Colisão com o mundo via mget/fget.
💡 Regra prática: use círculos para objetos que se movem rápido (projéteis), AABB para o jogador e inimigos, e tiles para o cenário. Misturar os três é o normal.
⚠️ Detecção e resposta são coisas separadas. Detectar que dois objetos se sobrepõem é fácil — decidir como afastá-los sem bugs é onde está o trabalho.
2

Colisão círculo × círculo

A técnica mais simples do Picotron. Dois círculos colidem quando a distância entre seus centros é menor que a soma dos raios. Para evitar a raiz quadrada (cara em CPU), compare os quadrados.

-- detecção: retorna true se os dois círculos colidem
function colide_circ(x1, y1, r1, x2, y2, r2)
  local dx = x2 - x1
  local dy = y2 - y1
  local dist2 = dx*dx + dy*dy
  local raios = r1 + r2
  return dist2 < raios*raios
end

-- uso no _update:
if colide_circ(px, py, 8, ex, ey, 10) then
  -- jogador tocou inimigo!
  vida -= 1
end

-- com lista de inimigos:
for e in all(inimigos) do
  if colide_circ(px, py, 8, e.x, e.y, e.r) then
    del(inimigos, e)
    pontos += 1
  end
end
✓ Sem raiz quadrada: comparar dist² com (r1+r2)² dá o mesmo resultado e é muito mais rápido. Use sempre essa forma.
💡 Para colisão círculo × ponto (ex: coletar um item sem tamanho): use colide_circ(px, py, r, item.x, item.y, 0) — raio 0 funciona perfeitamente.

Colisão rect × rect (AABB)

Dois retângulos sem rotação colidem quando se sobrepõem nos dois eixos ao mesmo tempo. A representação mais prática é com x, y (canto superior-esquerdo), largura w e altura h.

-- detecção AABB: x,y = canto sup-esq; w,h = tamanho
function colide_aabb(x1,y1,w1,h1, x2,y2,w2,h2)
  return x1 < x2+w2
     and x1+w1 > x2
     and y1 < y2+h2
     and y1+h1 > y2
end

-- uso: jogador 16x16 vs item 12x12
if colide_aabb(px,py,16,16, item.x,item.y,12,12) then
  sfx(0)
  del(itens, item)
end

-- guardar como tabela para reutilizar:
function bbox(obj)
  -- retorna o bounding box de um objeto
  return obj.x, obj.y, obj.w, obj.h
end

if colide_aabb(bbox(jogador), bbox(inimigo)) then
  -- colisão!
end
✓ AABB é a técnica mais usada para jogadores e inimigos. As 4 condições parecem muitas, mas o Lua avalia em curto-circuito — assim que uma for falsa, para.
💡 Dica de hitbox: faça o hitbox do jogador ligeiramente menor que o sprite visual (ex: sprite 16×16, hitbox 12×14). Isso torna o jogo mais justo e menos frustrante.

Colisão círculo × retângulo

Útil quando o jogador é um círculo e os obstáculos são caixas (ou vice-versa). A técnica usa o ponto mais próximo do retângulo ao centro do círculo.

-- cx,cy,cr = centro e raio do círculo
-- rx,ry,rw,rh = rect (canto sup-esq + tamanho)
function colide_circ_rect(cx, cy, cr, rx, ry, rw, rh)
  -- ponto mais próximo do rect ao centro do círculo
  local px = mid(cx, rx, rx+rw)
  local py = mid(cy, ry, ry+rh)
  
  -- distância do centro do círculo a esse ponto
  local dx = cx - px
  local dy = cy - py
  
  return dx*dx + dy*dy < cr*cr
end

-- uso: bala circular vs obstáculo retangular
for obs in all(obstaculos) do
  if colide_circ_rect(bala.x, bala.y, 4,
                      obs.x, obs.y, obs.w, obs.h) then
    del(balas, bala)
    obs.vida -= 1
  end
end
✓ A função mid(v, min, max) do Picotron faz o clamp perfeitamente — ela retorna o valor do meio entre os três, que é exatamente o que precisamos para encontrar o ponto mais próximo.

Colisão com o mapa (tiles)

Para colidir com o cenário, você consulta as células do mapa via mget() e verifica o flag de solidez com fget(). Tiles marcados com flag 0 são sólidos.

-- tamanho do tile em pixels (padrão Picotron = 16)
local TS = 16

-- retorna true se o pixel (x,y) está dentro de um tile sólido
function solido(x, y)
  local cx = flr(x / TS)
  local cy = flr(y / TS)
  return fget(mget(cx, cy), 0)
end

-- testa os 4 cantos do retângulo do jogador
function colide_mapa(x, y, w, h)
  return solido(x,     y)
      or solido(x+w-1, y)
      or solido(x,     y+h-1)
      or solido(x+w-1, y+h-1)
end

-- no GFX Editor: selecione o tile de parede
-- e ligue o flag 0 (primeiro quadradinho)
mget(cx,cy)Índice do sprite na célula (cx,cy) do mapa
fget(n, f)Valor do flag f do sprite n — true se ligado
mset(cx,cy,n)Troca o tile da célula — útil para blocos destrutíveis
flr(x/16)Converte coordenada de pixel em índice de célula
⚠️ Lembre de marcar os tiles sólidos no GFX Editor antes de testar. Um tile sem flag 0 ligado não vai colidir, mesmo sendo visível na tela.
💡 Você pode usar outros flags (1–7) para outros significados: flag 1 = perigoso (espinhos), flag 2 = escada, flag 3 = água. Assim o mapa vira uma descrição completa do mundo.

Resposta: separação de eixos

Detectar colisão não basta — você precisa responder: mover o objeto para fora do obstáculo. A técnica mais sólida é separar os eixos: mova X e resolva, depois mova Y e resolva. Isso permite deslizar ao longo de paredes.

function _update()
  -- 1. calcular velocidade desejada
  vx, vy = 0, 0
  if btn(0) then vx = -2 end
  if btn(1) then vx =  2 end
  vy += 0.3  -- gravidade

  -- 2. mover no eixo X e resolver colisão X
  px += vx
  if colide_mapa(px, py, pw, ph) then
    px -= vx   -- desfaz só o movimento X
    vx = 0
  end

  -- 3. mover no eixo Y e resolver colisão Y
  py += vy
  if colide_mapa(px, py, pw, ph) then
    py -= vy   -- desfaz só o movimento Y
    vy = 0    -- zera a velocidade vertical (pousa no chão)
  end
end

-- checar se está no chão (para liberar o pulo)
function no_chao()
  return solido(px,      py+ph)
      or solido(px+pw-1, py+ph)
end
✓ A separação de eixos é a razão pela qual o jogador consegue deslizar ao longo de uma parede em vez de travar. Se você mover X e Y juntos e reverter os dois ao colidir, o personagem para completamente.
⚠️ Se a velocidade em um frame ultrapassar o tamanho do tile (16px), o objeto pode atravessar paredes. Limite a velocidade máxima: vy = min(vy, 12).

Jogo de plataforma completo

Juntando tudo: mapa com tiles sólidos, jogador com AABB, gravidade, pulo, e colisão por separação de eixos. Cole no main.lua e desenhe alguns tiles sólidos no GFX/mapa:

local TS = 16

function solido(x, y)
  return fget(mget(flr(x/TS), flr(y/TS)), 0)
end

function colide_mapa(x, y, w, h)
  return solido(x,     y)     or solido(x+w-1, y)
      or solido(x,     y+h-1) or solido(x+w-1, y+h-1)
end

function no_chao()
  return solido(px, py+ph) or solido(px+pw-1, py+ph)
end

function _init()
  px, py = 32, 32
  pw, ph = 12, 14  -- hitbox menor que o sprite
  vx, vy = 0, 0
  
  -- inimigos como tabela de objetos
  inimigos = {}
  add(inimigos, {x=200, y=100, r=8, dir=1})
  add(inimigos, {x=320, y=150, r=8, dir=-1})
end

function _update()
  -- input
  vx = 0
  if btn(0) then vx = -2 end
  if btn(1) then vx =  2 end
  if btnp(4) and no_chao() then vy = -4 end
  
  -- gravidade (com velocidade máxima)
  vy = min(vy + 0.3, 8)
  
  -- mover + resolver X
  px += vx
  if colide_mapa(px, py, pw, ph) then px -= vx; vx = 0 end
  
  -- mover + resolver Y
  py += vy
  if colide_mapa(px, py, pw, ph) then py -= vy; vy = 0 end
  
  -- inimigos: patrulha + colisão círculo×círculo
  for e in all(inimigos) do
    e.x += e.dir
    if e.x < 16 or e.x > 460 then e.dir = -e.dir end
    
    local cx = px + pw/2
    local cy = py + ph/2
    local dx = cx - e.x
    local dy = cy - e.y
    if dx*dx + dy*dy < (e.r+8)*(e.r+8) then
      del(inimigos, e)
      sfx(0)
    end
  end
end

function _draw()
  cls(1)
  map()
  for e in all(inimigos) do
    circfill(e.x, e.y, e.r, 8)
  end
  rectfill(px, py, px+pw-1, py+ph-1, 12)
  print("inimigos: "..count(inimigos), 10, 10, 7)
end
✓ Este código combina as três técnicas: colisão AABB com o mapa (cenário), colisão círculo×círculo com inimigos, e separação de eixos para a resposta física.
