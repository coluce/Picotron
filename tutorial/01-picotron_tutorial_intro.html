O que é o Picotron?

O Picotron é uma Fantasy Workstation criada pela Lexaloffle (os mesmos autores do PICO-8). É uma estação de trabalho retrô fictícia — um pequeno sistema operacional completo voltado para criar jogos pixel art, animações, músicas e demos.

Toda a programação é feita em Lua 5.4. Os projetos são chamados de cartridges (.p64) e podem ser exportados como HTML ou executáveis.

💡 Se você já conhece PICO-8, o Picotron é o seu "irmão mais velho": mesma filosofia, mas mais poderoso, com tela 16:9 e uma IDE completa.
📥 Adquira em: lexaloffle.com/picotron.php — disponível para Windows, Mac e Linux.

Estrutura de um cartridge

Ao abrir o Picotron, você trabalha com um cartridge — uma pasta virtual em /ram/cart/. O arquivo principal sempre se chama main.lua.

Todo jogo em Picotron tem três funções especiais que o sistema chama automaticamente:

function _init()
  -- Chamada UMA VEZ ao iniciar o jogo
  -- Use para: criar variáveis, carregar dados
end

function _update()
  -- Chamada 60x por segundo
  -- Use para: lógica, física, input do jogador
end

function _draw()
  -- Chamada a cada frame (60fps)
  -- Use para: desenhar tudo na tela
end
⚠️ Nunca coloque lógica de jogo pesada dentro de _draw(). Separe: lógica → _update(), desenho → _draw().

Para rodar o cartridge, pressione CTRL+R. Para voltar ao editor, pressione ESC.

CTRL+RExecutar / reiniciar o cartridge
ESCParar e voltar ao editor
CTRL+SSalvar o cartridge
save nomeSalvar como nome.p64 (no terminal)

Olá mundo: desenhando na tela

Vamos conhecer as funções gráficas básicas. Cole isso no main.lua e pressione CTRL+R:

function _init()
  x = 240  -- posição X (centro da tela)
  y = 135  -- posição Y (centro da tela)
end

function _update()
end

function _draw()
  cls(1)              -- limpa a tela com cor 1 (azul escuro)
  circfill(x, y, 20, 8) -- círculo sólido: x, y, raio, cor
  print("Ola, Picotron!", 180, 10, 7) -- texto: msg, x, y, cor
  rectfill(50, 200, 150, 220, 11) -- rect: x1,y1,x2,y2,cor
end

Funções gráficas essenciais:

cls(cor)Limpa a tela com uma cor. Sempre a primeira chamada em _draw()
print(txt,x,y,cor)Escreve texto na posição (x,y)
pset(x,y,cor)Pinta um pixel
line(x0,y0,x1,y1,cor)Desenha uma linha
rect(x0,y0,x1,y1,cor)Retângulo (borda)
rectfill(x0,y0,x1,y1,cor)Retângulo sólido
circ(x,y,r,cor)Círculo (borda)
circfill(x,y,r,cor)Círculo sólido
💡 As cores são números de 0 a 63. As 16 primeiras (0–15) são as mais comuns e iguais às do PICO-8. Use 7 para branco e 0 para preto.

Capturando input do jogador

O Picotron usa btn(n) para ler botões e btnp(n) para detectar apenas o primeiro frame de pressão. Vamos mover nosso círculo:

function _init()
  x = 240
  y = 135
  vel = 3  -- velocidade de movimento
end

function _update()
  if btn(0) then x -= vel end  -- ← esquerda
  if btn(1) then x += vel end  -- → direita
  if btn(2) then y -= vel end  -- ↑ cima
  if btn(3) then y += vel end  -- ↓ baixo
  
  -- Manter dentro da tela (480x270)
  x = mid(10, x, 470)
  y = mid(10, y, 260)
end

function _draw()
  cls(1)
  circfill(x, y, 10, 8)
  print("use as setas!", 190, 10, 7)
end
btn(0)← Seta esquerda
btn(1)→ Seta direita
btn(2)↑ Seta cima
btn(3)↓ Seta baixo
btn(4)Z ou C (botão de ação)
btn(5)X ou V (botão de ação 2)
btnp(n)Verdadeiro apenas no primeiro frame pressionado
mid(a,b,c)Retorna o valor do meio — útil para clamping
💡 x -= vel é shorthand Picotron para x = x - vel. O Picotron suporta +=, -=, *=, /=.

Sprites e o editor gráfico

Sprites são desenhados no GFX Editor — o segundo workspace (ícone de paleta no topo). Cada sprite ocupa uma posição numerada (1, 2, 3…).

Depois de desenhar um sprite no editor, use spr() para desenhá-lo:

function _draw()
  cls(1)
  
  -- spr(índice, x, y)
  spr(1, x, y)
  
  -- Com flip horizontal (útil para personagens)
  spr(1, x, y, false, true)  -- hflip=false, vflip=true
  
  -- Sprite maior (2x2 tiles)
  spr(1, x, y, false, false, 2, 2)
end

O tamanho padrão de um sprite é 16×16 pixels (diferente do PICO-8 que usa 8×8).

spr(n,x,y)Desenha sprite de índice n na posição x,y
spr(n,x,y,hf,vf)Com flip horizontal (hf) e/ou vertical (vf)
spr(n,x,y,hf,vf,w,h)w e h em tiles (ex: 2,2 = sprite 32×32)
palt(cor, transp)Define se uma cor é transparente (true/false)
💡 Por padrão, a cor 0 (preto) é transparente nos sprites. Use palt(0, false) para torná-la opaca, ou palt(n, true) para tornar outra cor transparente.
⚠️ Sempre desenhe algo no GFX Editor (workspace 2) antes de tentar usar spr(). Um sprite vazio é invisível mas não dá erro.

Jogo completo: "pega o ponto"

Vamos montar um jogo simples do zero. O jogador (círculo azul) deve tocar o alvo (círculo vermelho) para pontuar. Cole no main.lua:

function _init()
  -- jogador
  px, py = 240, 135
  vel = 3
  
  -- alvo
  novo_alvo()
  
  -- placar
  pontos = 0
  tempo  = 600  -- 10 segundos a 60fps
  estado = "jogando"
end

function novo_alvo()
  -- posição aleatória na tela
  tx = rnd(440) + 20
  ty = rnd(230) + 20
end

function _update()
  if estado == "jogando" then
    -- mover jogador
    if btn(0) then px -= vel end
    if btn(1) then px += vel end
    if btn(2) then py -= vel end
    if btn(3) then py += vel end
    px = mid(10, px, 470)
    py = mid(10, py, 260)
    
    -- colisão: distância entre jogador e alvo
    local dx = px - tx
    local dy = py - ty
    if dx*dx + dy*dy < 20*20 then
      pontos += 1
      sfx(0)   -- toca sfx 0 (opcional)
      novo_alvo()
    end
    
    -- contagem regressiva
    tempo -= 1
    if tempo <= 0 then
      estado = "fim"
    end
    
  elseif estado == "fim" then
    if btnp(4) then  -- Z para reiniciar
      _init()
    end
  end
end

function _draw()
  cls(1)
  
  if estado == "jogando" then
    -- desenhar alvo
    circfill(tx, ty, 10, 8)   -- vermelho
    circ(tx, ty, 10, 7)       -- borda branca
    
    -- desenhar jogador
    circfill(px, py, 8, 12)   -- azul claro
    
    -- HUD
    print("pontos: "..pontos, 10, 10, 7)
    print("tempo: "..flr(tempo/60), 380, 10, 7)
    
  elseif estado == "fim" then
    print("FIM DE JOGO!", 190, 120, 8)
    print("pontos: "..pontos, 210, 140, 7)
    print("[Z] para jogar de novo", 170, 160, 6)
  end
end
💡 Conceitos usados neste jogo: variáveis de estado ("jogando"/"fim"), detecção de colisão por distância, temporizador com contagem em frames, e reinício via _init().
⚠️ A linha sfx(0) só funciona se você tiver criado um som no SFX Editor. Pode remover essa linha se preferir.
