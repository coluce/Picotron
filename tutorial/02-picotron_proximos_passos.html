Desenhando mapas com map()

O Picotron tem um editor de mapas integrado (workspace do mapa, ícone de grade). Você desenha tiles usando os sprites do GFX Editor e depois renderiza tudo com uma linha de código.

O arquivo de mapa padrão é map/0.map e usa os sprites de gfx/0.gfx. Para desenhá-lo na tela:

function _draw()
  cls(1)
  
  -- map() básico — desenha o mapa inteiro
  map()
  
  -- map(cx, cy, sx, sy, cw, ch)
  -- cx,cy = célula inicial do mapa
  -- sx,sy = posição na tela
  -- cw,ch = quantidade de células a desenhar
  map(0, 0, 0, 0, 30, 17)
  
  -- desenhar o jogador POR CIMA do mapa
  circfill(px, py, 8, 12)
end
map()Renderiza o mapa inteiro na posição (0,0) da tela
map(cx,cy,sx,sy,cw,ch)Renderiza uma região: célula (cx,cy) → tela (sx,sy), tamanho cw×ch células
mget(cx,cy)Retorna o índice do sprite na célula (cx,cy) do mapa
mset(cx,cy,n)Define o sprite da célula (cx,cy) como n — útil para blocos destrutíveis
fget(n,f)Retorna o valor do flag f do sprite n — use para marcar tiles sólidos
💡 No GFX Editor, cada sprite tem 8 flags (0–7) que você pode ligar/desligar. Convenção comum: flag 0 = sólido (colide). Use fget(mget(cx,cy), 0) para checar colisão com o mapa.
⚠️ O tile padrão é 16×16 px. Para calcular a célula a partir de coordenadas de pixel: cx = flr(px/16), cy = flr(py/16).

Câmera para seguir o jogador

Quando o mapa é maior que a tela (480×270), você precisa mover a câmera para que o jogador fique sempre visível. A função camera() translada todo o sistema de coordenadas do desenho.

function _draw()
  cls(1)
  
  -- câmera centralizada no jogador
  -- subtrai metade da tela para centralizar
  camera(px - 240, py - 135)
  
  -- tudo desenhado depois usa as coords do mundo
  map()
  circfill(px, py, 8, 12)
  
  -- resetar câmera antes de desenhar o HUD!
  camera()
  print("pontos: "..pontos, 10, 10, 7)
end

Para uma câmera suavizada (sem cortes bruscos), use interpolação:

function _init()
  px, py = 100, 100
  cam_x, cam_y = px, py  -- posição atual da câmera
end

function _update()
  -- mover jogador...
  
  -- câmera suave: aproxima 20% por frame
  cam_x += (px - cam_x) * 0.2
  cam_y += (py - cam_y) * 0.2
end

function _draw()
  cls(1)
  camera(cam_x - 240, cam_y - 135)
  map()
  circfill(px, py, 8, 12)
  camera()  -- resetar para o HUD
end
💡 Sempre chame camera() (sem argumentos) antes de desenhar o HUD — caso contrário, os textos e barras de vida vão "viajar" junto com o mundo.

Áudio: efeitos e música

O Picotron tem um sintetizador de 64 nós e um tracker de 8 canais. Os efeitos sonoros são criados no SFX Editor (workspace de notas musicais) e tocados com sfx(). Músicas de fundo usam music().

function _init()
  -- iniciar música de fundo (padrão: loop infinito)
  music(0)
end

function _update()
  -- tocar efeito sonoro ao pressionar Z
  if btnp(4) then
    sfx(0)          -- sfx índice 0
  end
  
  -- sfx em canal específico (evita cortar outro)
  if coletou_item then
    sfx(1, 1)       -- sfx 1 no canal 1
  end
  
  -- parar música
  if game_over then
    music(-1)        -- -1 para a música
  end
end
sfx(n)Toca o efeito sonoro de índice n
sfx(n, canal)Toca sfx n no canal específico (0–7)
sfx(-1)Para todos os sfx
music(n)Inicia a trilha n em loop
music(-1)Para a música de fundo
✓ Dica de workflow: crie seus sfx com SPD baixo (1–4) para efeitos curtos e percussivos. SPD alto (16+) funciona melhor para melodias.
⚠️ O Picotron tem 8 canais de áudio. Cada sfx() ocupa um canal. Se não especificar o canal, o sistema escolhe um disponível automaticamente.

Múltiplos arquivos com include

Conforme o jogo cresce, é boa prática separar o código em arquivos .lua diferentes. Use include no topo do main.lua:

-- main.lua
include "player.lua"
include "inimigos.lua"
include "hud.lua"

function _init()
  player_init()    -- definida em player.lua
  inimigos_init()  -- definida em inimigos.lua
end

function _update()
  player_update()
  inimigos_update()
end

function _draw()
  cls(1)
  map()
  player_draw()
  inimigos_draw()
  hud_draw()      -- definida em hud.lua
end

Exemplo de como fica o player.lua:

-- player.lua
function player_init()
  px, py = 240, 135
  pvida = 3
end

function player_update()
  if btn(0) then px -= 3 end
  if btn(1) then px += 3 end
end

function player_draw()
  spr(1, px, py)
end

Para criar um novo arquivo de código: clique em [+] na aba do editor de código, digite o nome (ex: player) e o Picotron adiciona .lua automaticamente.

💡 Variáveis declaradas sem local são globais e visíveis em todos os arquivos incluídos. Use isso a seu favor para compartilhar estado, mas com moderação.

Tabelas Lua para listas e objetos

Em Lua, tabelas ({}) servem tanto como arrays quanto como objetos. São essenciais para gerenciar listas de inimigos, projéteis e itens.

-- lista de inimigos
inimigos = {}

function criar_inimigo(x, y)
  add(inimigos, {
    x = x, y = y,
    vida = 3,
    vel = 1
  })
end

function _update()
  -- iterar todos os inimigos
  for ini in all(inimigos) do
    ini.x += ini.vel
    
    -- remover se saiu da tela
    if ini.x > 500 then
      del(inimigos, ini)
    end
  end
end

function _draw()
  cls(1)
  for ini in all(inimigos) do
    spr(2, ini.x, ini.y)
  end
end
add(t, v)Adiciona o valor v ao final da tabela t
del(t, v)Remove o valor v da tabela t (seguro dentro de loops)
all(t)Iterador para usar em for x in all(t) — safe para del()
#tQuantidade de elementos na tabela t
count(t)Alternativa a #t, mais confiável com tabelas esparsas
✓ Use for x in all(t) sempre que for chamar del() dentro do loop. O iterador all() é seguro para remoção; um for i=1,#t normal não é.

stat() e técnicas de debug

A função stat() retorna informações do sistema em tempo real — muito útil para monitorar performance e debugar.

function _draw()
  cls(1)
  map()
  
  -- painel de debug (remova no jogo final!)
  camera()  -- resetar câmera antes
  print("cpu: "..flr(stat(1)*100).."%", 400, 250, 7)
  print("fps: "..stat(7), 400, 260, 7)
  print("obj: "..count(inimigos), 400, 240, 7)
end

Valores de stat(n) mais usados:

stat(0)Memória usada (em KB)
stat(1)CPU usada no último frame (0.0 a 1.0+)
stat(7)FPS atual
stat(32..35)Posição e botões do mouse (32=x, 33=y, 34=botões)
stat(36)Roda do mouse (scroll)

Outro recurso útil: o terminal do Picotron (ESC no desktop). Você pode inspecionar variáveis globais digitando o nome delas diretamente.

💡 Se stat(1) ficar acima de 1.0 consistentemente, o jogo está processando mais do que consegue em 60fps — hora de otimizar loops ou reduzir objetos na tela.
⚠️ O Picotron tem um limite de 8M instruções Lua por segundo. Loops muito pesados em _update() podem fazer o framerate cair para 30fps automaticamente.

Exportar e compartilhar seu jogo

Quando o jogo estiver pronto, o Picotron permite exportar como HTML, executável (Windows/Mac/Linux) ou como cartridge PNG para o BBS.

-- No terminal do Picotron:

-- 1. Salvar o cartridge primeiro
save meujogo.p64

-- 2. Capturar a label (capa do jogo)
-- Pressione CTRL+7 enquanto o jogo roda

-- 3. Adicionar metadados
about /ram/cart

-- 4. Exportar como HTML (roda no navegador)
export meujogo.html

-- 5. Exportar como PNG cartridge (256k)
export meujogo.p64.png

-- 6. Exportar como executável
export meujogo.bin
save nome.p64Salva o cartridge no drive do Picotron
export nome.htmlExporta como página web — fácil de hospedar no itch.io
export nome.p64.pngCartridge PNG de 256k — compatível com o BBS do Lexaloffle
export nome.binExecutável standalone para Windows, Mac ou Linux
CTRL+7Captura um screenshot do jogo rodando como capa do cartridge
about /ram/cartDefine título, autor e versão do cartridge
✓ O export HTML é o mais prático para compartilhar — funciona no itch.io, no BBS e em qualquer hospedagem estática. O jogo roda direto no navegador sem instalar nada.
💡 Recursos da comunidade: lexaloffle.com/bbs/?cat=13 (cartridges), lexaloffle.com/dl/docs/picotron_manual.html (manual oficial).
